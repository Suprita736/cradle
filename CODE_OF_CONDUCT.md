# Code of Conduct

## Purpose

Cradle is a community-driven repository built around experimentation, learning, and collaboration. This Code of Conduct establishes the expectations for everyone participating in the project and helps create a welcoming, respectful, and productive environment for contributors of all experience levels.

Whether you are reporting an issue, submitting a pull request, improving documentation, reviewing code, or participating in discussions, you are expected to follow these guidelines.

The goal of this Code of Conduct is to:

- Foster an inclusive and respectful community.
- Encourage constructive collaboration.
- Promote professional and friendly communication.
- Help contributors resolve disagreements respectfully.
- Maintain a safe environment for learning and experimentation.

---

## Scope

This Code of Conduct applies to all interactions within the Cradle community, including but not limited to:

- GitHub Issues
- Pull Requests
- Code Reviews
- Discussions
- Documentation Contributions
- Project Suggestions
- Community Conversations related to this repository

All contributors, maintainers, reviewers, and community members are expected to follow this Code of Conduct while participating in the project.

## Community Standards

Cradle welcomes contributors from different backgrounds, skill levels, and experiences. Every contributor plays a role in creating a positive and collaborative environment.

We encourage all community members to:

- Treat everyone with respect and professionalism.
- Be patient and supportive when helping others.
- Welcome constructive feedback and different perspectives.
- Focus on collaboration rather than competition.
- Assume positive intent during discussions.
- Appreciate contributions of all sizes, including documentation improvements and bug reports.
- Encourage learning by sharing knowledge and providing helpful guidance.

---

## Expected Behavior

Contributors are expected to:

- Communicate respectfully in issues, pull requests, and discussions.
- Keep technical discussions focused and constructive.
- Accept feedback gracefully and respond professionally.
- Review code objectively without personal criticism.
- Respect project decisions made by maintainers.
- Help create an inclusive environment where everyone feels comfortable contributing.
- Report security concerns or inappropriate behavior responsibly instead of escalating conflicts publicly.

## Unacceptable Behavior

To maintain a respectful and welcoming community, the following behaviors are not acceptable:

- Harassment, discrimination, or personal attacks.
- Hate speech or offensive language targeting any individual or group.
- Trolling, intimidation, or deliberately provoking conflicts.
- Sharing another person's private or confidential information without permission.
- Spam, misleading information, or intentionally disruptive behavior.
- Plagiarism or submitting work that violates intellectual property rights.
- Repeatedly ignoring project guidelines or maintainer decisions.
- Any behavior that creates an unsafe, hostile, or unwelcoming environment.

Contributors engaging in unacceptable behavior may be subject to moderation or further action by the maintainers.

---

## Contributor Responsibilities

Every contributor is responsible for helping maintain a healthy and collaborative community.

Contributors should:

- Follow this Code of Conduct during all project interactions.
- Read and follow the project's documentation before contributing.
- Respect feedback from maintainers and fellow contributors.
- Keep discussions constructive and focused on improving the project.
- Submit original work or properly acknowledge external resources when appropriate.
- Report bugs, documentation issues, and security concerns responsibly.
- Help new contributors by encouraging respectful communication and knowledge sharing.

By participating in the project, contributors agree to uphold these responsibilities and contribute positively to the community.

## Maintainer Responsibilities

Project maintainers are responsible for fostering a welcoming, fair, and collaborative environment.

Maintainers are expected to:

- Apply this Code of Conduct consistently and fairly.
- Review issues and pull requests respectfully.
- Provide constructive feedback to contributors.
- Encourage inclusive discussions and knowledge sharing.
- Address inappropriate behavior promptly and objectively.
- Take reasonable action to maintain a safe and productive community.
- Make moderation decisions in the best interest of the project and its contributors.

Maintainers may remove comments, reject contributions, or take other appropriate actions if behavior violates this Code of Conduct.

---

## Respectful Communication

Healthy communication is essential for successful collaboration.

When interacting with other community members:

- Be respectful, courteous, and professional.
- Focus feedback on ideas and code rather than individuals.
- Be open to different viewpoints and constructive criticism.
- Ask questions politely and provide clear explanations when offering suggestions.
- Avoid inflammatory language, personal attacks, or unnecessary arguments.
- Resolve disagreements through respectful discussion whenever possible.

A collaborative and supportive approach helps create a positive experience for everyone involved in the project.

## Reporting Guidelines

If you experience or witness behavior that violates this Code of Conduct, please report it to the project maintainers as soon as possible.

When submitting a report, include:

- A clear description of the incident.
- Links to the relevant issue, pull request, or discussion (if applicable).
- Any supporting screenshots or evidence that may help explain the situation.
- The approximate date and time of the incident.
- Any additional context that may assist the maintainers in reviewing the report.

All reports will be handled respectfully and reviewed as objectively as possible. Maintainers will make reasonable efforts to protect the privacy of individuals involved while investigating reported incidents.

Community members are encouraged to report concerns in good faith and avoid public arguments or harassment while an issue is under review.

---

## Enforcement Process

Maintainers are responsible for evaluating reported incidents and determining an appropriate response based on the severity and context of the behavior.

Possible actions may include:

- Providing a friendly reminder about community expectations.
- Requesting that inappropriate behavior be corrected.
- Removing comments or content that violates this Code of Conduct.
- Limiting participation in project discussions.
- Temporarily or permanently restricting contributions in cases of repeated or serious misconduct.

Enforcement decisions will be made with the goal of maintaining a respectful, welcoming, and collaborative environment for all contributors.

Contributors are expected to cooperate with maintainers during the review process and respect the final decisions made regarding Code of Conduct violations.

## Consequences

Violations of this Code of Conduct may result in actions considered appropriate by the project maintainers based on the severity and frequency of the behavior.

Possible outcomes include:

- Informal guidance or clarification of community expectations.
- A formal warning.
- Removal of inappropriate comments, issues, or pull requests.
- Temporary suspension from participating in project discussions or contributions.
- Permanent restriction from contributing to the project in cases of repeated or severe misconduct.

The objective of these actions is to maintain a safe, respectful, and collaborative environment for everyone rather than to punish contributors unnecessarily.
